### requirements.txt

**语法格式** 在`requirements.txt`文件中，可以只写第三方模块名称，不指定版本号，默认安装最新版本。

如果使用注释，使用`#`加文字内容。

**模块名称与版本号之间可以使用如下符号：**

-   `==`：等于
-   `>`：大于版本
-   `>=`：大于等于
-   `<`：小于版本
-   `<=`：小于等于版本
-   `~=`：兼容版本，使用任何大于或等于指定版本，但不大于当前发行系列的版本，例如`~=1.4.3`可以匹配`1.4.3`到`1.4.9`，但是不能匹配`1.5.0` 。

在可选范围内，安装最新版本，当前最优先的依旧是等于的版本号。

可以使用逗号`,`分隔两个条件，**例如下述代码：**
```
package_name >= 1.0, <=2.0
```
